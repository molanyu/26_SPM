# Tests DevOps Module Spec

## 1. 任务定位

本文件定义 `tests` 与 `devops` 在 `G9` 阶段的实现边界。

本阶段负责提供以下基础能力：

- 统一测试入口
- 关键模块自动化测试收口
- 最小 CI 流水线
- 测试失败即阻断流水线

## 2. 实现范围

本次实现只包含以下内容：

- 统一 `pytest` 测试入口
- 单元测试与集成测试的最小分层
- 对关键模块测试命令的稳定封装
- 一条可执行的 CI 工作流
- 基础测试说明文档

本次实现不包含以下内容：

- 自动部署到云端
- 蓝绿发布
- 性能压测流水线
- 安全扫描平台接入
- 多环境发布审批流

### 2.1 关键模块覆盖范围

本阶段必须确保以下模块已纳入自动化测试收口：

- `identity`
- `resource`
- `system_config`
- `reservation`
- `checkin`
- `violation`
- `notification`
- `assistant`

`admin_portal` 可纳入页面与权限基础测试，但不要求在本阶段扩展到完整端到端 UI 自动化。

### 2.2 流水线目标

第一版流水线只解决一个目标：

- 在代码变更后，自动执行项目关键测试并明确返回通过/失败结果

## 3. 测试规则

测试规则固定如下：

### 3.1 测试入口规则

- 项目必须存在统一测试命令
- 统一测试命令必须可在本地直接执行
- 统一测试命令必须可被 CI 直接复用

### 3.2 测试组织规则

- 测试按模块组织
- 每个已交付模块至少保留一组可独立执行的测试
- 新增测试不得破坏已有模块测试入口

### 3.3 测试通过规则

- 流水线中任一关键测试失败，CI 必须失败
- 不允许把失败测试标记为通过
- 不允许用跳过核心测试来伪造绿色流水线

### 3.4 环境规则

- 测试环境必须尽量独立、可重复
- 测试不得依赖人工交互
- 测试默认不依赖真实外部通知通道
- 真实 SMTP 通道只允许在显式配置的 smoke/manual 验证中启用

## 4. DevOps 规则

DevOps 规则固定如下：

### 4.1 第一版 CI 规则

- 第一版只建立一条测试流水线
- 流水线至少完成：
  - 安装依赖
  - 运行统一测试命令
  - 输出成功或失败结果

### 4.2 配置规则

- CI 配置文件必须版本化管理
- 流水线命令必须与本地测试命令保持一致
- 不允许在 CI 中使用与本地不同的隐藏测试逻辑

### 4.3 失败处理规则

- 测试失败即标记流水线失败
- 不自动忽略失败模块
- 不自动降级核心测试集

## 5. 模块边界

- `tests/devops` 是横切收口模块，不承载业务规则
- `tests/devops` 可以依赖各模块公开接口与测试夹具
- `tests/devops` 不直接修改业务模块公开语义
- `tests/devops` 不新增业务数据模型

跨模块协作规则固定如下：

- 如需补测试，优先补在对应模块的 `tests/<module>/` 下
- 如需统一入口或流水线配置，放在项目级测试与 CI 位置
- `tests/devops` 不通过修改业务规则来“修绿”流水线

## 6. 文件范围

本阶段不新增业务数据模型。

本阶段主要涉及：

- 项目级测试入口文件
- 模块测试文件
- CI 配置文件
- 测试说明文档

## 7. 文件级实现清单

本阶段实现至少包含以下文件或同等职责文件：

```text
tests/
  conftest.py
  identity/
  resource/
  system_config/
  reservation/
  checkin/
  violation/
  notification/
  assistant/

.github/workflows/test.yml

docs/devops/testing_and_ci.md
```

说明：

- 若当前仓库不使用 GitHub Actions，可使用等价 CI 配置文件，但必须只保留一套主配置

## 8. 实现顺序

实现顺序固定如下：

1. 统一本地测试命令
2. 收口关键模块测试
3. 确认测试环境可重复执行
4. 建立最小 CI 工作流
5. 建立测试与流水线说明文档

未完成前一步，不进入后一步。

## 9. 测试范围

本阶段至少覆盖以下验证：

- 统一测试命令可在本地执行
- 关键模块测试可全部执行
- CI 工作流可成功触发测试
- 测试失败时 CI 正确失败
- 无需真实通知通道即可完成测试
- 默认 CI 不连接真实 SMTP 服务

## 10. 完成标准

`tests/devops` 阶段完成时，必须满足：

- 项目存在统一测试入口
- 关键模块测试已纳入自动化执行
- 存在一条最小可运行 CI 流水线
- 测试失败会阻断流水线
- 测试与流水线说明文档可读
