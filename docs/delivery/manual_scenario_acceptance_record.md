### 3.4 SCN-01 正常履约流程

- [ ] 学生成功创建正常履约预约。
- [ ] 预约前提醒第一次执行包含正常预约 ID。
- [ ] 预约前提醒第二次执行为空，幂等通过。
- [ ] 正常预约通知日志只有一条 `RESERVATION_REMINDER`。
- [ ] 当前动态签到码可查看。
- [ ] 学生使用签到码签到成功。
- [ ] 正常预约状态变为 `CHECKED_IN`。
- [ ] 管理端预约记录可看到该 `CHECKED_IN` 预约。
- [ ] 管理端统计体现正常使用时长。
- [ ] 正常履约流程没有新增违约。

### 3.5 SCN-02 未签到违约与座位释放

- [ ] 学生成功创建未签到预约。
- [ ] 未签到提醒第一次执行包含未签到预约 ID。
- [ ] 未签到提醒第二次执行为空，幂等通过。
- [ ] 未签到通知日志只有一条 `NO_SHOW_REMINDER`。
- [ ] 超时释放第一次执行包含未签到预约 ID。
- [ ] 超时释放第二次执行为空，幂等通过。
- [ ] 未签到预约状态变为 `EXPIRED`。
- [ ] 同一未签到预约只生成一条违约记录。
- [ ] 未签到座位释放后重新可用。
- [ ] 管理端预约记录可看到该 `EXPIRED` 预约。
- [ ] 管理端违约记录可看到该学生未签到违约。
- [ ] 管理端统计体现违约数量。

### 3.6 通知通道

- [ ] `mock` 通道下通知日志状态正常。
- [ ] 如使用 `smtp_email`，预约前提醒邮件已收到。
- [ ] 如使用 `smtp_email`，未签到提醒邮件已收到。
- [ ] 重复执行同一提醒任务不会重复生成成功通知。
